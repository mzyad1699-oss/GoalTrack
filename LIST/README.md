# GoalTrack — المرحلة الأولى

تطبيق لمتابعة أهدافك الأسبوعية، مبني بـ HTML/CSS/JavaScript (بدون Framework)، ويستخدم **Supabase** للحسابات وقاعدة البيانات، مع **IndexedDB** كـ Cache محلي يخليك تشتغل حتى لو النت اتقطع لحظيًا.

## ✅ اللي موجود في المرحلة دي
- تسجيل حساب بالبريد الإلكتروني + تسجيل دخول بجوجل + نسيت كلمة المرور.
- كل مستخدم له بياناته الخاصة (محمية بـ Row Level Security في Supabase).
- إضافة/تعديل/حذف الأهداف الثابتة (مع نقاط لكل هدف).
- عرض الأسبوع (يبدأ من السبت) كـ"بوصلة رحلة" + مسار أيام تفاعلي.
- لكل يوم: Checkbox، ملاحظات، وقت التنفيذ، أولوية (منخفضة/متوسطة/عالية).
- تحديد يوم كـ"يوم راحة" بدون التأثير على النسبة.
- حساب النقاط والنسبة تلقائيًا لحظة بلحظة.
- رسم بياني بسيط لنقاط الأسبوع (Chart.js).
- ملاحظات نهاية الأسبوع (أفضل/أسوأ شيء، تحدي، تطوير، ملاحظات عامة).
- الوضع الداكن/الفاتح.
- العمل Offline مؤقتًا مع مزامنة تلقائية عند عودة النت.

## 🚀 خطوات التشغيل

### 1) إعداد Supabase
1. اعمل حساب على [supabase.com](https://supabase.com) وأنشئ مشروع جديد (مجاني).
2. من **SQL Editor** شغّل الملف `sql/schema.sql` بالكامل — ده هيعمل كل الجداول والحماية.
3. من **Project Settings → API** انسخ:
   - `Project URL`
   - `anon public key`
4. افتح `js/supabaseClient.js` وحط القيمتين مكان:
   ```js
   const SUPABASE_URL = 'YOUR_SUPABASE_PROJECT_URL';
   const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
   ```

### 2) تفعيل تسجيل الدخول بجوجل (اختياري)
من **Authentication → Providers → Google** في لوحة Supabase، فعّلها وحط بيانات مشروعك من Google Cloud Console (Client ID / Secret)، وضيف رابط الموقع بتاعك في Redirect URLs.

### 3) تشغيل المشروع محليًا
المشروع عبارة عن ملفات ثابتة، لازم تتفتح عن طريق سيرفر محلي (مش بفتح الملف مباشرة) عشان الـ ES Modules تشتغل صح. مثلًا:
```bash
npx serve .
# أو
python3 -m http.server 5500
```
وبعدين افتح المتصفح على العنوان اللي هيظهر لك.

## 📁 هيكل المشروع
```
goaltrack/
├── index.html
├── css/style.css
├── js/
│   ├── supabaseClient.js   # إعداد الاتصال بـ Supabase
│   ├── auth.js             # تسجيل دخول / خروج / حسابات
│   ├── database.js         # طبقة البيانات + Cache بـ IndexedDB
│   ├── utils.js            # دوال مساعدة (تواريخ، حسابات)
│   ├── components.js       # مكونات الواجهة (البوصلة، مسار الأيام...)
│   ├── charts.js           # الرسوم البيانية
│   └── app.js              # التحكم الرئيسي وربط كل حاجة
└── sql/schema.sql          # جداول قاعدة البيانات + الحماية
```

## 🗺️ خارطة المراحل القادمة
المرحلة دي هي الأساس (Architecture) اللي كل حاجة تانية هتتبني فوقه:
- **المرحلة 2:** نظام الإنجازات (Achievements) + المستويات (XP/Level) + الـ Streak + العملات.
- **المرحلة 3:** Heatmap سنوي + Calendar View تفاعلي + البحث والفلاتر.
- **المرحلة 4:** تقارير أسبوعية/شهرية/ربع سنوية/سنوية تلقائية + مقارنة الأداء.
- **المرحلة 5:** صفحة Explore لمشاركة Templates + نظام Export/Import + Auto Backup.
- **المرحلة 6:** ربط AI Review (OpenAI/Gemini/Claude API) لتحليل الأداء.
- **المرحلة 7:** الرسوم البيانية المتقدمة (Pie/Radar/مقارنة أسابيع) + صفحة إحصائيات كاملة + ثيمات إضافية.

قولّي لما تكون جاهز تكمل أي مرحلة من دول وهنبنيها فوق نفس الأساس ده.
